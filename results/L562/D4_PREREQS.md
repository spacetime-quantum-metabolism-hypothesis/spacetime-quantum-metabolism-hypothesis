## §1. 4 조건 protocol *방향*

### 조건 1 — 유일성 no-go protocol

**목표**: D4 도출에서 SK 측도 인자가 "유일하게 강제되는지" 사전 차단.

**방향 (수식 없음)**:
- (a) **대안 인자 후보 enumeration**: SK 경로적분 측도에서 정규화 인자로 들어올 수 있는 모든 후보를 *먼저 카탈로그화* 한다. 카탈로그는 도출 *이전* 에 작성되어야 하며, 도출 후 후보 추가 금지.
- (b) **도출 경로 다중화**: 동일 도출 목표에 대해 SK 경로 *외* 의 독립 출발점(예: 응답함수 정규화, 분배함수 위상, 측도 유일성 정리 등)을 병렬로 시도. 동일 인자가 모든 경로에서 강제되어야 "유일성" 인정.
- (c) **음성 통제 (negative control)**: 카탈로그의 *틀린* 후보를 채택했을 때 어떤 내부 모순(unitarity, hermiticity, 차원, 부호)이 발생하는지 *각 후보마다* 사전 기록. 모든 비-1/(2π) 후보가 명시 모순을 일으켜야 통과.
- (d) **외부 심판**: 카탈로그/음성 통제 표를 SK 측도 도출과 무관한 4인 (예: open-system / Lindblad / KMS 전공자) 에게 blind 검증.

**통과 기준 (방향)**: 카탈로그의 모든 비-채택 후보가 (c) 의 모순 진단으로 격리되고, (b) 의 독립 경로 ≥ 2 개에서 동일 인자가 재현될 때만 "유일성 no-go" 성립.

### 조건 2 — postdiction 외형 완화 protocol

**목표**: L556 → L560 시간 순서가 "결과를 알고 도출했다" 의심을 받는 외형을 완화.

**방향 (수식 없음)**:
- (a) **Blind derivation 재실행**: D4 도출 작업을, 관측 타깃값(a₀ 등)을 *마스킹* 한 채 새 팀(이전 라운드 미참여) 에게 위임. 도출 결과가 마스킹 해제 후 관측과 일치할 때만 "blind 성공".
- (b) **Pre-registration archive**: D4 도출에 사용되는 출발 가정 / 채택 측도 / 정규화 선택을 도출 *착수 전* 에 timestamp 봉인 (git tag 또는 외부 archive). 도출 후 가정 변경 금지.
- (c) **결과 미인지 시뮬레이션**: 도출 팀에게 a₀ 의 *섭동된 가짜 타깃값* 을 함께 제공해 어느 것이 진짜인지 모르는 상태로 도출. 도출이 어느 가짜와도 무관하게 자체 동역학으로 종결되어야 함.
- (d) **외형 vs 실질 분리 보고**: 시간 순서상 postdiction 외형은 인정하고, (a)~(c) 로 *실질적* postdiction 위험만 차단했음을 명시.

**통과 기준 (방향)**: (b) 가 갖춰지고 (a) 또는 (c) 중 하나가 시연되면 "외형 완화" 성립. (a)/(c) 모두 불가능한 경우 외형 유지 + 한계 명시.

### 조건 3 — pillar 격리 protocol

**목표**: 4-pillar 중 SK pillar *단독* 으로 D4 가 도출되는지, 다른 3 pillar 와의 cross-talk 가 결과를 끌어주는지 분리.

**방향 (수식 없음)**:
- (a) **Pillar 단독 도출**: 다른 3 pillar (각각의 가정/결과) 를 *전혀 참조하지 않은* 상태에서 SK 만으로 도출. 참조 차단은 도출 팀의 입력 자료 자체에서 물리적으로 분리.
- (b) **Cross-pillar 의존도 그래프**: 도출 단계마다 어떤 pillar 의 어떤 명제가 사용되었는지 dependency graph 작성. SK 외 pillar 노드가 등장하면 그 단계는 "non-isolated" 표시.
- (c) **Leave-one-out 재도출**: 4 pillar 중 SK 외 한 pillar 를 제거한 상태에서 D4 가 여전히 도출되는지 4 회 반복. 단 1 회라도 도출이 깨지면 SK 단독성 미충족.
- (d) **Cross-validation 분리 보고**: SK 단독 결과 vs 4-pillar 결합 결과를 별도 섹션으로 보고. "SK 단독 강도" 와 "4-pillar 합치 강도" 를 혼동 금지.

**통과 기준 (방향)**: (a) 도출이 (b) 그래프상 SK 노드 only 로 닫히고, (c) leave-one-out 4/4 통과 시 격리 성립.

### 조건 4 — falsifier 등록 protocol

**목표**: D4 priori 가 a₀ 외 어떤 관측에 의해 falsified 되는지 사전등록.

**방향 (수식 없음, 관측 카테고리만 열거)**:
- (a) **RAR universality 위반**: 은하 종족 간 RAR 산포가 D4 가 허용하는 한계 이상으로 커지는 경우.
- (b) **Cluster vs galaxy a₀ 차이**: 은하단 스케일에서 추론된 가속도 임계가 은하 스케일과 통계적으로 유의하게 다를 때.
- (c) **a₀ 의 z-진화**: 고적색편이에서 a₀ 가 시간 의존성을 보일 때 (D4 가 정적 측도를 강제한다면 falsify).
- (d) **환경 의존성**: a₀ 가 국소 밀도/포텐셜 환경에 의존할 때 (universal measure 가정 위반).
- (e) **Wide-binary / 위성은하 dynamics**: 외부장 효과 or 가속도 임계 구조가 D4 예측과 어긋날 때.
- (f) **사전등록 형식**: 위 (a)~(e) 각각에 대해 *어떤 데이터셋 / 어떤 통계량 / 어떤 임계치* 가 falsifier 로 작용하는지 도출 *이전* 에 명시. 사후 falsifier 추가 금지.
- (g) **Severity 등급**: hard-kill (한 건이면 D4 폐기) vs soft-tension (재검증 트리거) 분리 등록.

**통과 기준 (방향)**: (a)~(e) 중 ≥ 3 개가 (f) 의 형식으로 사전등록되고, ≥ 1 개가 hard-kill 등급으로 명기되면 통과.
