## §2 부활 가능성 등급 정리

- **medium (P3a)**: paradigm 의 K-Z 재해석 경로가 가장 직접적. 외부 framework 회피 protocol 을 명시적으로 통과시킬 수 있다면 priori 회복 가능.
- **medium-low (RG)**: anchor 재해석 방향은 열림. 단, truncation 의존성은 paradigm 무관 별개 박탈 사유로 잔존 — 이 채널까지 별도 해소 필요.
- **low (D4, D2)**: 박탈 핵심이 *수학적 선택* 이 아니라 *선택 시점의 시간순서* 에 있음. paradigm 은 수학 구조를 재해석할 뿐, 선택 history 를 되돌리지 못함.

---

## §3 paradigm 의 진정 효과

paradigm shift 가 가져오는 *진정한* 변화:

1. **L549/L552 부활 trigger 가능**:
   - K-Z 의 시간성을 count-rate 로 재해석 (L549) — 외부 import 의 본질이 바뀜.
   - RG anchor 를 count anchor 로 재해석 (L552) — circularity 의 회로가 끊길 가능성.
   - 두 경로 모두 paradigm 에 의해 박탈 사유 *프레임 자체* 가 흔들림.

2. **부활 ≠ 자동 회복**:
   - paradigm 은 *프레임 변화* 를 제공할 뿐, 부활은 외부 framework 회피 protocol (P3a) 또는 truncation 독립성 입증 (RG) 등 *별도 절차* 통과 후 가능.
   - 본 매트릭스는 trigger 가능성 표지이며 회복 선언이 아님.

---

## §4 R3 시간순서 위반은 paradigm 도 회피 못함

- **L562/L566 박탈 그대로**:
  - 박탈 사유의 본질이 *수학 선택의 정당성* 이 아니라 *결과 사전 인지 후 선택* 이라는 시간순서.
  - paradigm 은 SK measure 의 의미와 Z₂ wall 의 위상을 재해석할 수 있으나, "1/(2π) 를 알고 있었다" 는 인식 history 는 변경 불가.
  - 따라서 paradigm 적용 후에도 D4/D2 는 R3 default FAIL 상태 유지.

- **R3 시간순서는 paradigm-invariant**:
  - 어떤 paradigm 도 인식 사건의 시간순서를 되돌리지 못함.
  - cherry-pick 판정 중 *선택 시점* 에 기반한 항목은 paradigm 면역.
